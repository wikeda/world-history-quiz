## 1. 目的・ターゲット

- **メインユーザー**: 大学受験生（世界史B / 探究の入試対策）
- **ゴール**: 用語の想起力を高め、得点につなげる
- **基本方針**: 硬派・効率重視をベースに、「飽きずに続けられる」仕掛けを上乗せする
- **デバイス**: スマホ前提（モバイルファースト。片手・スキマ時間で回せること）

---

## 2. 全体方針（コンセプト）

自己採点フラッシュカードで高速に周回しつつ、**「繰り返すほど習熟度が上がる」ことを可視化**して継続を促す。
差別化の核は次の3点:

1. **4方向スワイプ**の回答体験（下方向の"絞り"による答え合わせ）
2. **章ごとの積層棒グラフ**（繰り返すほど緑=マスターが積み上がる）
3. **習熟マップ**（27章を色で俯瞰、100%で🏆コンプリート）

---

## 3. 技術構成

- **フロントエンド**: Vite + React
- **状態/保存**: ブラウザの `localStorage`（サーバーなし・ログインなし）
- **公開**: GitHub Pages（静的サイト。Vite の `base` をリポジトリ名に設定）
- **PWA**: ホーム画面に追加／オフライン学習（問題データを同梱、初回読込後は電波不要）
- **スタイル**: CSS 変数ベース（テーマ・アクセントカラーの着せ替えを低コストで実現）
- **依存**: 軽量に保つ。グラフは基本 inline SVG で自作（必要に応じ軽量チャートライブラリを検討）

> 当面はローカルGitで管理し、GitHub 連携・Pages 公開は後日設定する。

## 4. データモデル

### 4.1 問題データ（静的・同梱）

出典は Google スプレッドシート。列構成は以下のシンプルな4列:

| 列 | 内容 |
|---|---|
| No | 通し番号 |
| チャプター | 章名（27章） |
| 質問 | 問題文 |
| 解答 | 答え |

- 全27章・約847問。1章あたり平均約31問。
- ビルド前に **JSON へ変換して同梱**する（例: `questions.json`）。
- **データ補正（要対応）**: `No.206`（マグナ・カルタ）はチャプター列が空欄。正しくは「中世ヨーロッパ」。取り込み時に補完する。

問題1件のイメージ:

```json
{ "no": 4, "chapter": "古代オリエント", "question": "シュメール人が粘土板に刻んで使用した文字を何というか", "answer": "楔形文字" }
```

### 4.2 進捗データ（localStorage）

問題ごとの学習状態と、章・全体の集計、ストリーク、設定を保持する。

- 問題ごと: 状態、`覚えた連続回数`、最終学習日時、（章グラフ用の）セッション履歴
- 章ごと: 各状態の個数（マスター/あやふや/できなかった/未回答）、習熟度%、繰り返し回数、伸びグラフ用の時系列
- 全体: 全体習熟度%、コンプ章数、ストリーク（現在/最長/学習日ログ）
- 設定: テーマ、アクセントカラー、出題順、セッション問題数、マスター判定、通知、効果音/振動

> localStorage は端末・ブラウザ依存のため、機種変更やデータ削除で消える。対策として **進捗の JSON エクスポート/インポート**（バックアップ）を用意する。

### 4.3 習熟度の状態機械（4状態）

各問題は次の4状態のいずれかを持つ:

| 状態 | 色 | 意味 |
|---|---|---|
| 未回答 | 灰 `#c9ccd6` | まだ一度も出題していない |
| できなかった | 赤 `#e5736b` | 未定着。最優先で再出題 |
| あやふや | 黄 `#e6b800` | もう一歩。近いうちに再出題 |
| マスター | 緑 `#3f9e5a` | 定着（習熟度%にカウント） |

遷移ルール（回答＝スワイプ結果）:

- **右＝覚えた**: `覚えた連続回数` +1。**連続2回でマスターに昇格**。
- **上＝あやふや**: 状態→あやふや。`覚えた連続回数`を0にリセット。
- **左＝できなかった**: 状態→できなかった。`覚えた連続回数`を0にリセット。
- 「覚えた1回のみ（あと1回でマスター）」の中間段階は、**グラフ/内訳上は当面「あやふや(黄)」に含めて扱う**。
  - （将来オプション）5色目「あと1回（薄緑）」として分離表示も可能。

> マスター判定は既定「覚えた連続2回」。設定で「1回で即マスター」に緩められる。

## 5. 学習ロジック

### 5.1 回答方式：4方向スワイプ（自己採点フラッシュカード）

問題を見て頭の中で答える → 答え合わせ → 自己判定、という流れ。

- **右スワイプ = 覚えた**（習熟度アップ／マスターへ前進）
- **左スワイプ = できなかった**（未定着へ・最優先で再出題）
- **上スワイプ = あやふや**（近いうちに再出題）
- **下ドラッグ = 解答を徐々に表示**（バカラの"絞り"。引くほど鮮明に。**判定ではなく答え合わせ**。離しても表示は残る）
- 判定は閾値を超えて離した時点で確定 → 次のカードへ。
- **PC対応（副次）**: `←/↑/→` キーで判定、`↓` キーで絞り表示。
- 演出・閾値の重さは設定/実装時に微調整（絞りの演出、効果音、振動）。

### 5.2 セッション構成

- 1セッションの問題数は設定で選択（**10 / 20 / 30 / 制限なし**、**既定は「制限なし」**）。
- 出題順は設定の**出題順トグル（順番 / ランダム）**に従う。
- セッション終了で**リザルト画面**へ。

### 5.3 出題対象（重要な確定事項）

- **「この章を学習」**: 未マスター（未回答＋できなかった＋あやふや）を中心に出題し、**マスター済みもたまに混ぜて**定着確認する（マスターは出題頻度を下げる）。
- **「あやふや◯問だけ」**: いま「あやふや」状態の問題のみを狙い撃ちする短時間の復習。
- **「もう一度挑戦」（リザルト）**: その章をもう一周（＝上記「この章を学習」と同じ通常出題）。繰り返し学習の主線。
- **「できなかった◯問だけ復習」（リザルト）**: 直前セッションで落とした問題だけ。

## 6. 画面設計

### 6.1 ナビゲーション：3タブ

フッターのメインナビは **ホーム / 成績 / 設定** の3タブ。
（当初あった「章」タブは、ホーム＝習熟マップが章一覧を兼ねるため廃止。）

- 表示は**アイコンのみ**（🏠 / 📊 / ⚙️、ラベルなし・大型）。選択中はアイコン上にアクセントバー＋非選択は淡色。
- フッターが表示される画面: ホーム・成績・設定
- フッターを隠す没入フロー: **出題中・リザルト・章詳細**（戻るは左上「‹」）

### 6.2 ホーム（習熟マップ）

- 上部: タイトル、🔥ストリーク、**全体習熟度リング**（%＋「◯/◯問マスター」＋「🏆コンプ章 ◯/27」）
- **⚡あやふや復習**（件数バッジ付き）への近道
- **27章のヒートマップ**（4列グリッド。タイル＝章名＋習熟度%。色は習熟度ランプ。100%は🏆）
- 出題順トグル（順番/ランダム）
- タイルタップで章詳細へ

### 6.3 成績

- 全体習熟度リング＋マスター数
- ストリーク（現在/最長）
- **全27章の習熟度リスト**（スクロール）。色ドット＋章名＋%（100%は🏆）、タップで章詳細へ。
  - 並び順は **「苦手順（習熟度が低い順・既定）／章順」をタップで切替**。
- （後追い）今週カレンダー・全体習熟度の週別推移グラフは未実装。

### 6.4 設定

- **表示**: テーマ（ライト/ダーク/端末に合わせる）、アクセントカラー（青/緑/紫/橙/ピンク/ティール）
- **学習**: 出題順、1セッションの問題数、マスター判定（2回連続↔1回）、毎日のリマインダー
- **サウンド・触覚**: 効果音、スワイプ時の振動
- **データ**: 進捗エクスポート、進捗インポート、進捗リセット
- アプリ情報/バージョン

### 6.5 章詳細 ＋ 伸びグラフ

- ヘッダー: 戻る、章名、問題数、習熟度%（100%は🏆＋金色バッジ）
- 現在の内訳バー（マスター/あやふや/できなかった/未回答の個数）
- **この章の伸び（積層棒グラフ）** … 6.6 参照
- 出題順トグル
- アクション: 「⚡あやふや◯問だけ」／「▶この章を学習」（100%時は「🏆コンプリート達成！」／「🔁もう一度ぜんぶ復習」）

### 6.6 積層棒グラフ（この章の伸び）

- **横軸**: 学習の回数（1回目〜N回目、セッションごとに右へ点/棒が増える）
- **縦軸**: 問題数。**母数＝章の全問で固定 → 各回とも棒の高さは一定**
- **積層（下から）**: マスター(緑) / あやふや(黄) / できなかった(赤) / 未回答(灰)
- 回を重ねるほど**緑が下から伸び、赤・灰が消えていく**
- **各棒の緑の上端に、白文字で習熟度%を直接表示**（チップ・フチなし。緑の高さ＝マスター率＝%で意味が一致）
- 100%時は全面緑＋白文字「100%」
- （エッジ）緑がごく小さい回は白文字を小さく or 省略で対応

### 6.7 出題画面（フラッシュカード）

- 上部: 戻る、章名、進捗（7/30）、セッション進捗バー
- カード: 問題文 → 下ドラッグで解答が"絞り"表示
- 4方向スワイプで判定（5.1）
- フッター非表示（没入）

### 6.8 リザルト（お疲れ様）

- **ライト基調**（アプリ全体と統一。ダークはテーマ設定で全画面切替）
- 🎉＋「おつかれさま！」＋章名・問題数
- **紙吹雪 8つ**（色・角度・位置をばらけさせる。※4は不可）
- 🔥ストリーク更新（増えた場合は +1 表示）
- 内訳（覚えた/あやふや/できなかった）
- **この章の習熟度の変化**（before→after、例 17%→20% +3%）
- **実際の積層棒グラフを埋め込み表示**（この章の伸びをその場で確認。※「グラフに記録しました」の文言は廃止）
- アクション（優先順）: **🔁もう一度挑戦** → ⚡できなかった◯問だけ復習 → ホームにもどる
- ※あやふや復習（全章）から来た場合は章名の代わりに「あやふや復習」表示、章%・グラフは非表示、再挑戦は同復習へ

## 7. モチベーション設計

- **ストリーク（連続学習日数）**: 毎日の起動を促す
- **習熟マップ**: 27章を色で俯瞰、弱点が一目・埋めたくなる収集欲
- **章ごとの積層棒グラフ**: 「繰り返すほど伸びる」を体感（本アプリの目玉）
- **100%コンプリート＝🏆トロフィー**: 実績的な達成感
- 循環: 学習 → リザルトで「+7%」実感 → 章詳細グラフで積み上げを眺める → また学習

## 11. 実装で確定した事項（旧・未決事項）

- "たまにマスター"の混ぜる比率＝**未マスター数の 20%**（`maintenanceRatio: 0.2`）で確定。
- セッション問題数の既定＝**制限なし**で確定。
- 問題データ更新フロー：スプレッドシートをCSVエクスポート → `data/questions.csv` に配置 → `npm run questions` で再生成（README参照）。
- グラフの横軸「N回目」＝**章を「この章を学習」した回数（セッション単位）**で確定。あやふや復習(全章)はチャプター周回に含めない。
- round記録の off-by-one（最後の1問の反映漏れ）は実装時に修正済み（回答反映後の描画で記録）。
