# 世界史 一問一答

スマホ向けの世界史一問一答学習アプリ。**4方向スワイプ**で高速に周回しながら、**習熟度の伸びを可視化**して「繰り返すほど定着する」を実感できる。ログイン不要・オフライン対応・進捗は端末内に保存。

> **ステータス:** ✅ 公開中
> **🔗 https://wikeda.github.io/world-history-quiz/**
> スマホで開いて「ホーム画面に追加」するとアプリのように使えます（オフライン対応）。

---

## 特長

- **4方向スワイプ回答**（スマホ最適・片手操作）
  - → 覚えた / ← できなかった / ↑ あやふや / ↓ ドラッグで解答を"絞り"表示（バカラの絞りのイメージ）
  - PCでは矢印キー（←↑→で判定、↓で解答表示）に対応
- **4段階の習熟度**（未回答 / できなかった / あやふや / マスター）。「覚えた」を連続2回でマスター（設定で1回に変更可）
- **習熟マップ**：全27章を習熟度カラーで俯瞰、章名＋%表示、100%達成で 🏆 トロフィー
- **章ごとの積層棒グラフ**：学習を重ねるほど緑（マスター）が積み上がる伸びを可視化。リザルト画面にも埋め込み、その場で伸びが見える
- **出題ロジック**：未マスター中心＋たまにマスターを混ぜて定着確認。「あやふや◯問だけ」「できなかった◯問だけ」のピンポイント復習も
- **ストリーク**（連続学習日数）でモチベ維持
- **着せ替え**：ライト/ダーク/端末連動テーマ＋アクセントカラー選択
- **PWA**：ホーム画面に追加、電波なしでも学習可能
- **進捗の管理**：JSON エクスポート/インポート（バックアップ）／リセット

## 画面構成（3タブ）

| タブ | 内容 |
|---|---|
| ホーム | 習熟マップ（27章のヒートマップ・章名＋%）・全体習熟度・あやふや復習 |
| 成績 | 全体習熟度・ストリーク・全27章の習熟度リスト（苦手順/章順で切替） |
| 設定 | テーマ/アクセント・出題順・セッション問題数・マスター判定・データ管理 |

下部ナビはアイコンのみ（🏠 / 📊 / ⚙️）。このほか、章詳細（伸びグラフ）／出題（フラッシュカード）／リザルトは、ナビを隠す没入画面。

## 技術構成

- **Vite + React + TypeScript**（静的SPA）
- ルーティング: **React Router（HashRouter）** … GitHub Pages でのリロード404を回避
- 状態/保存: React Context + **localStorage**（サーバー・ログインなし）
- グラフ: 依存を増やさず **inline SVG** で自作
- テスト: **Vitest + Testing Library**（ドメインロジックはTDD、全38テスト）
- PWA: **vite-plugin-pwa**（オフラインキャッシュ＋インストール可能）
- CI/CD: **GitHub Actions** で `main` への push ごとに自動ビルド＆Pages公開

## 開発

```bash
npm install
npm run dev        # 開発サーバ（http://localhost:5173/world-history-quiz/ で開く）
npm test           # テスト
npm run questions  # 問題CSV→JSON変換（data/questions.csv を配置後）
npm run build      # 本番ビルド（tsc -b && vite build）
```

### 問題データ

出典は Google スプレッドシート（`No / チャプター / 質問 / 解答` の4列・全27章・約847問）。
`ファイル > ダウンロード > CSV` で書き出して `data/questions.csv` に配置し、`npm run questions` で `src/data/questions.json` を再生成する。取り込み時に空欄チャプターを直前の章で補完する。

---

## デプロイ（GitHub Pages / GitHub Actions）

`main` に push すると `.github/workflows/deploy.yml` が走り、自動でビルド＆公開される（手動作業なし）。

- 配信元: GitHub Pages の Source = **GitHub Actions**
- `vite.config.ts` の `base` はリポジトリ名（`/world-history-quiz/`）と一致
- PWA更新: 公開後、端末側は**一度リロード（または再起動）**すると新バージョンに切り替わる

---

## データの扱い

学習の進捗はすべて**ブラウザ内（localStorage）に保存**され、サーバーには送信されません。
機種変更やブラウザのデータ削除で消えるため、設定の**エクスポート**で定期的にバックアップできます。
